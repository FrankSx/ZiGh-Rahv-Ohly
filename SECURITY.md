# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please do NOT open a public issue.

Instead:
- Email the repository maintainer directly.
- Provide a detailed technical description.
- Include proof-of-concept code if applicable.

Responsible disclosure is expected.

## Cryptographic Evolution

If a primitive becomes deprecated:
- Open a proposal issue.
- Provide a formal reduction argument for replacement.
- Document migration path.
